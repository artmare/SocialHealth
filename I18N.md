# Internationalization

SocialHealth uses Flask-Babel for interface translations.

## Runtime setup

- Flask-Babel is installed through `requirements.txt`.
- The Babel extension is created in `app/extensions.py`.
- `create_app()` initializes it with a locale selector in `app/__init__.py`.
- Supported locales are `en`, `ru`, and `uk`.
- URLs can use locale prefixes: `/`, `/ru/...`, `/uk/...`.
- Compiled catalogs are stored in `app/translations/<locale>/LC_MESSAGES/messages.mo`.

## Marking strings

Use gettext wrappers in Python and Jinja:

```python
from flask_babel import lazy_gettext as _
flash(_("Settings saved"), "success")
```

```jinja2
{{ _("Settings") }}
```

Avoid hardcoded UI text in templates, route flash messages, form labels, and
client-side strings. Put strings behind `_()` and update catalogs.

## Commands

Extract strings:

```bash
python scripts/i18n.py extract
```

Update existing catalogs:

```bash
python scripts/i18n.py update
```

Add a new locale:

```bash
python scripts/i18n.py init <locale>
```

Compile catalogs:

```bash
python scripts/i18n.py compile
```

Validate catalogs before commit/deploy:

```bash
python scripts/i18n.py check
python -m pytest tests/test_i18n.py -q
```

Commit both `.po` and `.mo` files. Vercel uses the compiled `.mo` catalogs at
runtime, so they are intentionally tracked for `app/translations/**`.
